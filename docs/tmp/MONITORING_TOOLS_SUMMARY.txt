Three integrated monitoring tools that work together:

1. PIPELINE MONITOR DASHBOARD
   File: monitoring-agent pipeline_monitor
   Purpose: Displays overall pipeline health and status
   Shows:
     • Current pipeline status
     • Fetch timestamps and freshness
     • Database table row counts
     • Recent log activity
   Usage: ./.cache/bin/monitoring-agent pipeline_monitor
   API Calls: ZERO (reads from logs)

2. LIVE DELTA MONITOR
   File: monitoring-agent live_delta_monitor
   Purpose: Real-time user change tracking
   Shows:
     • Users changed in configurable time window (default 30s)
     • Data freshness indicators
     • Pending operations
     • Recent sync activity
   Usage: ./.cache/bin/monitoring-agent live_delta_monitor [SECONDS] [--compact]
   API Calls: ZERO (reads from logs)
   
   Strategy Implementation:
   • Uses range[updated_at] filtering (same as live_db_sync.sh)
   • Tracks changed users in time window
   • Updates three tables: users, projects_users, achievements_users
   • Window parametable (default 30 seconds)
   • Filters to kind=student only

3. BACKLOG HELPER
   File: toolkit-agent backlog_helper
   Purpose: Manage pending user syncs and track state
   Commands:
     • add USER_ID [REASON]       - Add user to backlog
     • list                        - Show pending users
     • status                      - Show statistics
     • clear                       - Clear backlog
     • process --test              - Simulate processing (no API calls)
   Storage:
     • logs/.monitor_state/pending_users.jsonl
     • logs/.monitor_state/processed_users.jsonl
   API Calls: ZERO in test mode (default)

./.cache/bin/toolkit-agent backlog_helper clear
