Sprint split (1-week sprints)
- Sprint 0
  - Freeze module set, architecture baseline, Definition of Done, branch strategy.
- Sprint 1
  - Frontend/backend frameworks, auth base, repo scaffolding cleanup.
- Sprint 2
  - User interaction module + OAuth + i18n baseline.
- Sprint 3
  - Realtime features + microservices integration.
- Sprint 4
  - Monitoring + health/status page + backups/DR + browser compatibility.
- Sprint 5
  - GDPR + stabilization + module demo scripts + README contribution mapping.
- Sprint 6 (buffer/stretch)
  - Security hardening and analytics/dashboard/design/search stretch modules.

Project management routine
- Weekly planning (PM leads, PO prioritizes, Tech Lead validates feasibility).
- Daily async update per member: yesterday/today/blocker.
- Pull request rule: at least 1 reviewer, Tech Lead required for critical paths.
- End-of-sprint demo with checklist against claimed modules.
